Since 2011, I have been working as software developer in globoesporte.com and sportv.com, both written in Python/Django. The web sites are, respectively, the first and second top-rated sports portals in Brazil. In these projects, in addition to the Back End, I also have developed the Front End, consolidating the knowledge of jQuery and CSS and Semantic Web, used for relationship content. In the two most recent projects (UFC Combate and Eu Atleta), were utilized technologies to improve page rank, such as using HTML5 and Schema.org tags. Between 2009 and 2011, I have developed the 2010 version of Cartola FC (cartolafc.globo.com), the fantasy game of SportTV. Cartola FC works with data about users and leagues that have hundreds, or even thousands, of users. It also has statistics about each player of the Brazilian soccer league. In each round, there is a process we call "market update" in which the players' scores are calculated, the users' scores are calculated according to the team he put together and finally the leagues are ranked. This process involves a database with more than 2GB of data. The challenge in 2010 version was to optimize this process, that used to take about 14 hours, to be held in just 4 hours. We used multiprocessing and multi threads to perform this optimization while optimizing the data structure and the python code. As we could develop the entire system from scratch, development of Cartola FC and also the process of market update were designed to achieve the performance goal. The result of this work was a market update performed in 40 min. With an architecture built with 100% free software and using Python language, the fully dynamic application broke paradigms in the company by proposing an innovative and robust, highly scalable and performative application (serves 2 million users), using Python, Tornado, nginx and MySQL. Moreover, the Front End was developed using jQuery and CSS. I developed several APIs to integrate with Cartola FC mobile application. In this project, I also have the opportunity to work with MongoDB, Redis, memcached and to study, superficially, the Hadoop. From 2005 to 2009, I developed several sites using Java for globo.com mobile area. Among them, a mobile portal (m.globo.com) in two versions: for iPhone and other devices. I have also participated in the development of the previous version (wap.globo.com), with recognition of phone model. The main technologies used were J2EE, JSP, XSL, Struts and Oracle database. Besides that, several recommendations for SEO (Search Engine Optimization) were used.
